Thanks for using gittodoistclone

The idea behind gittodoistclone is that the developer has created issues in GitHub repositories. The developer then wants to see those issues in todoist as a way to track them.

The preferred way to use gittodoistclone is for a developer to group his/her issues via GitHub milestones. gittodoistclone initially creates a project in todoist with a name that matches the repository name. For each milestone, that has issues gittodoistclone can create a subproject by the name of the milestone. The issues then are tasks in the todoist subproject.

Via the user interface the developer selects which issues from which milestones he/she wants cloned to todoist

If the developer does not want to do the above, gittodoistclone has an All category. Selecting from All> simply copies the tasks directly to the newly created todoist project
